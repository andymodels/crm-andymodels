AndyRadio — front estático do andymodels.com (sem backend no domínio do site)
================================================================================

Endpoints (mesmo JSON):
  GET https://SEU-CRM/api/radio
  GET https://SEU-CRM/api/public/radio
  GET https://SEU-CRM/api/public/radio/v2

Substitua SEU-CRM por PUBLIC_APP_URL do CRM (sem barra final).

Por playlist, usar no player:
  playlist_cover_url  (ou cover_url — duplicado) → imagem da capa da playlist
  curator_name        → texto de curadoria (se vazio, o site define o fallback)
  curator_instagram   → URL do link (se vazio, não linkar ou usar fallback)

Fetch no site (evitar cache antigo no browser):
  fetch(url, { cache: 'no-store', credentials: 'omit' })

O servidor CRM envia Cache-Control: no-store neste JSON.

Variável de build (exemplo):
  NEXT_PUBLIC_RADIO_JSON_URL=https://....onrender.com/api/public/radio/v2

Ver exemplo mínimo: public-radio-response.example.json

Helpers JS: radioPublicJson.js (playlistCoverArtUrl, curatorDisplay).

Não é necessário outro Express/API só para a rádio no andymodels.com.

Desativar remoção automática de playlists de teste na 1.ª subida do servidor CRM:

  RADIO_SKIP_PURGE_TEST_ON_BOOT=1
